# Методология генерации описаний товаров для techno-modern.ru

## 1. Анализ текущей ситуации

### Выявленные проблемы в описаниях

| Тип проблемы | Пример | Частота |
|-------------|--------|---------|
| Машинный перевод | "Домохозяйство электрического мяса из шлифования мяса" | ~60% |
| Упоминания Taobao | "Товар с Taobao. Продавец: xxx" | ~40% |
| Шаблонные описания | "Качественный товар из категории Строительство" | ~15% |
| Несоответствие товару | "Качественные автомобильные шины" для аппарата педикюра | ~5% |
| Бессвязный текст | "Текстильная ткань, папка, диван, отрывной лист" | ~20% |

## 2. Лучшие практики e-commerce копирайтинга 2024-2025

### Ключевые принципы

1. **Выгоды важнее характеристик**
   - Не "нержавеющая сталь" → "напитки остаются холодными 24 часа"
   - Не "мощность 280 Нм" → "справится с любыми болтами в автосервисе"

2. **Структура для сканирования**
   - Короткий вводный абзац (эмоциональная связь)
   - Буллеты для характеристик
   - Люди читают максимум 20-28% текста

3. **Оптимальная длина**
   - 150-300 слов (600-1500 символов)
   - Не более 2000 символов для маркетплейсов
   - Ключевые слова в первых 100 символах

4. **SEO-оптимизация**
   - Long-tail ключевые слова конвертят лучше
   - Писать для людей, не для роботов
   - Уникальный контент для каждого товара

### Структура идеального описания

```
[ЗАГОЛОВОК - уже есть в name]

[ВВОДНЫЙ АБЗАЦ - 1-2 предложения]
Ответ на вопрос "Зачем мне это нужно?"

[ОСНОВНЫЕ ПРЕИМУЩЕСТВА - 2-3 пункта]
• Выгода 1 (не характеристика!)
• Выгода 2
• Выгода 3

[ТЕХНИЧЕСКИЕ ХАРАКТЕРИСТИКИ - буллеты]
• Параметр: значение
• Параметр: значение

[ПРИМЕНЕНИЕ - опционально]
Для кого/для чего подходит
```

## 4. Промпт-инструкция для генерации описаний

### Универсальный промпт (базовый)

```
Ты — опытный копирайтер интернет-магазина бытовых товаров.

ВХОДНЫЕ ДАННЫЕ:
- Название товара: {product_name}
- Категория: {category_name}

ТРЕБОВАНИЯ К ОПИСАНИЮ:
1. Длина: 400-800 символов (без учёта буллетов)
2. Структура:
   - 1-2 предложения: для чего товар и главное преимущество
   - 3-4 буллета: выгоды (не просто характеристики!)
   - 2-4 буллета: технические характеристики (извлечь из названия)
3. Тон: информативный, дружелюбный, без воды
4. НЕ использовать:
   - "Качественный", "надёжный" без конкретики
   - Китайские названия и бренды (если не известные)
   - Слова "уникальный", "лучший", "идеальный"
   - Эмодзи и восклицательные знаки
5. Извлечь из названия:
   - Материал (если указан)
   - Размер (если указан)
   - Количество в комплекте (если указано)
   - Технические параметры (мощность, объём и т.д.)

ФОРМАТ ОТВЕТА (только это, без преамбул):
[Вводный текст 1-2 предложения]
```

## 5. Технический план batch-обработки

### Этап 1: Подготовка (1 категория = тестовая)

1. Выбрать категорию с наибольшим количеством проблемных описаний
2. Экспортировать 20 товаров для тестовой генерации
3. Сгенерировать описания
4. Провести ручную валидацию
5. Скорректировать промпт по результатам

### Этап 2: Пакетная обработка

```javascript
// Псевдокод процесса
async function generateDescriptions(categoryId) {
  // 1. Получить товары категории
  const products = await supabase
    .from('products')
    .select('id, name, description, category_id')
    .eq('category_id', categoryId)
    .limit(50); // батчами по 50

  // 2. Фильтровать товары с плохими описаниями
  const needsUpdate = products.filter(p =>
    p.description.includes('Taobao') ||
    p.description.includes('Качественный товар из категории') ||
    p.description.length < 100 ||
    hasGibberish(p.description)
  );

  // 3. Генерировать описания через LLM API
  for (const product of needsUpdate) {
    const newDescription = await generateWithLLM(product.name, categoryName);

    // 4. Валидация перед сохранением
    if (validateDescription(newDescription)) {
      await supabase
        .from('products')
        .update({ description: newDescription })
        .eq('id', product.id);
    }
  }
}
```

### Этап 4: Скрипт обновления

```sql
-- Выявить товары с плохими описаниями
SELECT id, name, description,
  CASE
    WHEN description ILIKE '%taobao%' THEN 'taobao_mention'
    WHEN description ILIKE '%качественный товар из категории%' THEN 'template'
    WHEN LENGTH(description) < 100 THEN 'too_short'
    WHEN description ~ '[А-Яа-я]+\s+[А-Яа-я]+\s+[А-Яа-я]+\s+[А-Яа-я]+\s+[А-Яа-я]+.*Товар' THEN 'gibberish'
    ELSE 'other'
  END as issue_type
FROM products
WHERE
  description ILIKE '%taobao%'
  OR description ILIKE '%качественный товар из категории%'
  OR LENGTH(description) < 100
  OR description ~ 'Продавец:';
```

## 6. Критерии качества описаний

### Автоматические проверки (обязательные)

| Критерий | Условие | Действие при нарушении |
|----------|---------|----------------------|
| Минимальная длина | >= 200 символов | Отклонить |
| Максимальная длина | <= 2000 символов | Обрезать |
| Упоминание Taobao | Отсутствует | Отклонить |
| Упоминание продавца | Отсутствует | Отклонить |
| Шаблонные фразы | Отсутствуют | Отклонить |
| Китайские символы | Отсутствуют | Отклонить |
| Структура | Есть абзац + буллеты | Предупреждение |

### Проверка на "мусор" (regex паттерны)

```javascript
const GIBBERISH_PATTERNS = [
  /([А-Яа-я]+\s+){5,}\s*(Товар|товар)/,  // повторяющиеся слова
  /подушка\s+подушка/i,
  /шейки\s+матки/i,
  /\b[A-Za-z]{10,}\b/,  // длинные английские слова (ошибки)
  /\d{10,}/,  // длинные числа (артикулы и т.д.)
];
```

### Ручная валидация (выборочная)

**Чек-лист для проверяющего:**

- [ ] Описание соответствует названию товара
- [ ] Нет грамматических ошибок
- [ ] Нет бессмысленных фраз
- [ ] Указаны характеристики из названия
- [ ] Выгоды понятны покупателю
- [ ] Нет невыполнимых обещаний
- [ ] Тон соответствует категории

### Метрики качества (для мониторинга)

1. **Процент отклонённых** — не более 10%
2. **Средняя длина описания** — 400-800 символов
3. **Уникальность** — нет дублей между товарами
4. **Соответствие структуре** — >= 90%
